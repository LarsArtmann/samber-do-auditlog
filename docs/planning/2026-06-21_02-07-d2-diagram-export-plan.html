05 Risks & Blockers

Risk: D2 output format assumptions

We assume go-output/d2 v0.13.0 emits valid D2 syntax with direction:, node id: label, and style.fill: directives. If the renderer changes output shape in a future version, tests will need updating. Mitigation: pin to v0.13.0, same as graph/plantuml/escape.

Risk: Coverage drop below 95%

New code paths (d2.go + tests) add ~20-30 lines of production code and ~80-100 lines of test code. Test code is excluded from coverage. The production code is small but the coverage denominator grows. Mitigation: the new WriteD2 method is fully exercised by tests; any drop will be <0.1%.

Quick win: depguard already allows go-output/*

The .golangci.yml depguard rule allows github.com/larsartmann/go-output as a prefix. No depguard configuration changes needed for the d2 submodule.

Quick win: zero new external dependencies

Scratch-module verification confirmed go-output/d2 compiles with only golang.org/x/sys and golang.org/x/term — both already present in the project's module graph. No go.mod bloat beyond the single d2 submodule line.
