### Core library (production-shaped)

- **Lifecycle auditing**: registration, invocation, shutdown, health-check event capture with timestamps, sequence numbers, phases (before/after), and per-event provider type.
- **Stack-based dependency inference**: `A` on-stack when `B`'s before-hook fires → `A→B`. Forward + reverse deps computed and sorted.
- **Provider type detection** via `do.ExplainNamedService`: `lazy` / `eager` / `transient` / `alias`, surfaced per-event and per-service.
- **Capability detection**: `IsHealthchecker` / `IsShutdowner` via `do.ExplainInjector` (called outside the recorder lock — deadlock-safe).
- **Health check auditing**: `RecordHealthCheck[WithContext](injector)` wraps `injector.HealthCheckWithContext()`, records `EventTypeHealthCheck` events, updates `ServiceInfo` health fields.
- **Service status state machine**: `registered` → `active` → `shutdown`, with error variants (`invocation_error`, `shutdown_error`). Computed via `deriveServiceStatus`.
- **Memory-bounded capture**: `Config.MaxEvents` cap + `Config.InitialEventCapacity` pre-sizing + `Plugin.DroppedEventCount()`.
- **Report validation**: `Report.Validate()` cross-checks denormalized count fields against actual data.
- **Report querying**: `ServiceByName`, `ServiceByRef`, `ServicesByScope`, `EventsByService`, `EventsByRef`, `EventsByType`, `FailedServices`, `UnhealthyServices`, `Index()`.
- **Report filtering**: `Report.Filtered(opts...)` with 5 options (`WithServicesByName`, `WithServicesByType`, `WithEventsByType`, `WithTimeRange`, `WithScope`) + pruned scope tree.
- **Schema migration**: `MigrateReport([]byte)` upgrades v0.1.0 JSON → current schema; rejects empty/missing-version input; preserves `ExportedAt`.
- **Real-time observability hook**: `Config.OnEvent` callback fired outside the mutex.
- **Zero-cost disabled mode**: empty hooks returned by `Opts()`; only samber/do's own ~115ns overhead.
- **Concurrent-safe**: single `sync.RWMutex` + 2 atomics; each hook acquires the lock exactly once.

### Exports

- **JSON report**, **NDJSON event stream**, **self-contained HTML** (dark amber "Container Telemetry" redesign with lifecycle waveform, 5-tab layout, Sugiyama DAG graph, dual timeline).
- **Mermaid flowchart** and **PlantUML component diagram** exports via shared `diagramFormatter` interface with theme directives.
- **Atomic file writes**: temp-file + `os.Rename` so a crash leaves the previous file intact.
- **Buffered I/O**: 64KB `bufio` blocks (10–100x fewer syscalls).
- **Zero-allocation struct map key** for `serviceKey`.

2. **Stop treating docs as an afterthought to code.** The CHANGELOG and TODO drift here was severe and entirely avoidable with a "update changelog in the same commit as the feature" discipline.
