1. **Strips `go.sum` changes** from the staged commit, resulting in empty commits (commit `fc0f899` was committed as empty). Fix: commit `go.sum` with `--no-verify`.

**Impact**: These are documented BuildFlow config issues (noted in go-output's own `AGENTS.md` gotchas) — not fixable in-repo. The workaround (`--no-verify` for go.sum and CoC commits) is brittle and easy to forget.

## E. WHAT WE SHOULD IMPROVE

### Architecture / Code Quality

1. **`ServiceInfo` is a god object** (18 fields) — it mixes identity, lifecycle, health, and graph data. The planned split into `ServiceIdentity`/`ServiceLifecycle`/`ServiceHealth`/`ServiceGraph` would make the model more honest and composable. This is the v0.1.0 breaking batch.
2. **`dedupGraphEdges` is a local duplicate of go-output's `DedupEdges`** — exists because the D2 renderer doesn't embed `GraphRendererState`. Filing an upstream PR would eliminate this.
3. **String-typed identifiers** — `ContainerID`, `ScopeID`, `ServiceName` are bare `string`. Named types would make impossible states unrepresentable (e.g., passing a service name where a scope ID is expected). Deferred to v0.1.0.

### Testing

4. **No property-based testing** — Diff/Filter/Migrate are pure functions over complex inputs; `rapid` or `gopter` would catch edge cases the unit tests miss.
5. **CSV/TSV not in example** — the self-checking demo doesn't exercise the delimited exports.

### CI / DevOps

6. **BuildFlow hook conflicts with go.sum and CoC** — the `--no-verify` workaround is brittle. Either reconfigure BuildFlow or document the exemption prominently.
7. **No pre-commit govulncheck** — currently CI-only; adding to the hook would catch vulnerabilities earlier.
8. **Node.js 20 deprecation warnings** in CI — actions/checkout@v4 and actions/setup-go@v5 target Node 20. Not failing yet but will eventually.

### Documentation

9. **README has dedicated sections for Mermaid, PlantUML, DOT, D2** — could be collapsed into a single "Diagram Export" section for less scrolling.
10. **No D2 output example in README** — Mermaid has a code block showing sample output; D2 should match (partially done — sample added this session).
